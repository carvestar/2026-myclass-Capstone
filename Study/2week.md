# 캡스톤디자인 ChatGPT 설명

## 개념
**캡스톤 디자인은 대학 교육의 마지막 단계에 수행하는 종합 설계 프로젝트 수업이다.**

주로 졸업 전 단계에서 수행하며 실무 역량 향상을 목표로 한다.

## 목적
- 전공지식의 실제 적용 능력 향상
- 팀 협업 및 의사소통 경험 습득
- 문제 해결 및 개발 프로세스 이해

## 진행 과정
주제 선정 -> 문제 정의 -> 설계 및 개발 -> 테스트 및 개선 -> 결과 발표

## 특징
- 팀 프로젝트 중심 수업
- 실제 서비스 또는 시스템 제작
- 산업 현장과 유사한 개발 경험 제공

# ChatGPT 소개 및 기능

## ChatGPT란?
ChatGPT는 자연어 기반 인공지능으로 질문 응답, 학습 보조, 코드 작성, 자료 조사 등을 지원하는 AI 도구이다.  
자기주도학습 및 캡스톤디자인 프로젝트 수행에 활용할 수 있다.

## 주요 기능
- **새 채팅** : 새로운 대화 시작
- **채팅 검색** : 이전 대화 기록 검색
- **이미지** : 이미지 생성 및 편집
- **앱(Apps)** : 외부 도구 연동 기능
- **심층 리서치** : 심화 조사 및 분석 수행
- **Codex** : 프로그래밍 및 코드 작업 지원
- **프로젝트** : 대화 및 작업 단위 관리

## 캡스톤디자인 활용 예시
- 프로젝트 아이디어 기획
- 기술 조사 및 자료 정리
- 코드 작성 및 디버깅
- 보고서 및 발표 자료 작성

# 1. 생성형 AI 시대의 LLM 활용서

## 1) 생성형 AI와 거대 언어 모델(LLM)
### 2016년 알파고 등장 이후 AI의 비약적 발전
- 생성형 AI 등장: 기존 데이터를 바탕으로 새로운 콘텐츠(텍스트, 이미지, 음악 등)를 만들어내는 기술
- 거대 언어 모델
  - 자연어 처리 분야(NLP)에서 엄청난 변화야기
  - 자연어 분야의 다양한 작업 수행 가능
      - **방대한 양의 데이터 학습 -> 문맥 이해, 창의적 글쓰기, 번역, 요약**
  - 정보 접근성 크게 향상
  - 개인 맞춤형 서비스 제공 가능

### 생성형 AI 주요 특징
- 콘텐츠 생성
  - **주어진 입력에 따라 새로운 텍스트, 이미지, 음악 생성**
- 창의성
  - **기존 데이터 학습 -> 새로운 아이디어나 콘텐츠 생성 능력**
- 응용 분야
  - **게임, 영화, 광고, 교육 등 여러 산업 사용**

### LLM (Large Language Model)
- 방대한 텍스트 데이터를 학습하여 문맥을 이해하
- 의미 있는 응답 생성에 뛰어난 능력
- 자연어 처리 특화
- LLM 주요 특징
  - 언어 이해
    - **문맥 파악 후 문법적으로 올바른 문장 생성 능력**
    - 대화형 AI, 번역기, 텍스트 요약기 등 다양한 응용 프로그램 활용
  - 대화 생성
    - **사용자의 질문이나 요청에 대해 자연스럽고 일관된 대화 생성 가능**
  - 지식 기반
    - **학습 데이터 바탕으로 다양한 주제에 대한 정보 제공 가능**

## 2) ChatGPT 첫 만남
### ChatGPT
- 거대 언어 모델의 대중화 이끌어 냄
  - 압도적 성능, 사용자 친화적 인터페이스
  - 광범위한 지식과 정보를 채팅이란 친숙한 대화 형식으로 제공 -> 혁신적 인공 지능 서비스
- GPT(Generative Pre-trained Trnasformer) 모델 기반
  - 사전 학습된 방대한 데이터 활용
  - 사용자 질문 답변
  - 정보 검색, 창작 지원, 문제 해결 등 다양한 분야에서 활발히 활용
 
### 다양한 거대 언어 모델 서비스
- Gemini, LLaMA, Claude, DeepSeek, Grok, CLOVA-X, wrtn

# 2. 소프트웨어 공학 소개

## 1) 소프트웨어(S/W) 이해
### 프로그램과 소프트웨어
- 프로그램: 원시코드
- 소프트웨어: 원시코드, 모든 산출물(자료구조, 테스트 결과 등), 단계마다 생성되는 문서, 사용자 매뉴얼
- 소프트웨어가 더 포괄적인 개념

### 소프트웨어의 특징
 - **개발 중심**: 제조가 아닌 창의적 개발이며 개인의 역량에 따라 결과물 차이가 큼
 - 소모가 아닌 **품질 저하**
   - H/W: 오래 사용 시 부품 닳음, 고장 발생 빈도 높음, 기능 떨어짐
   - S/W: 오새 사용해도 닳지 않음, 고장 발생 빈도 낮음, 기능 동일함
 - **실패 곡선**
   - H/W 실패 곡선
     - 초기 실패율 높음 -> 오류 해결 -> 오랜 기간 사용 -> 주변 환경 문제 발생(먼지, 진동) -> 실패율 증가
   - 이상 S/W 실패 곡선
     - 개발 완료 후 변경 사항/환경 변화 없어야 함
     - 발견되지 않은 오류로 초기 실패율 높음 -> 오류 해결 -> 오랜 기간 사용
   - 실제 S/W 실패 곡선
     - 초기 실패율 높음 -> 오류 해결 -> 실패율 낮음 -> 변경 발생(기능 추가 및 수정) -> 변경으로 인한 부작용 -> 실패율 급격히 증가
 - **소프트웨어 당면 과제**
   - 소프트웨어 개발의 느린 발전 속도
   - 새로운 소프트웨어에 대한 사용자 요구 증가 -> 해결 방안: CBD발 방법론
   - 관리 기술의 부분적 활용 -> 관리 필요
 - 소프웨어 개발의 어려움
   - 개발 과정 복잡
   - 참여 인력 다수
   - 개발 기간 긺
