# 5. 클로드 AI 개요와 활용
## 1) claude.ai 개요
### 클로드(Claude) AI claude.ai
- OpenAI에서 연구를 진행하던 연구자들이 2021년 설립: 다리오 아모데이
- "안전하고 정확한 AI"가 핵심 철학
  - AI 기술의 발전이 잠재적으로 가져올 위험성을 인식
  - 이러한 위험을 완화하고 AI가 인간에게 신뢰할 수 있는 도구로 자리 잡도록하는 것을 목표
  - AI 기술이 보다 윤리적이고 신뢰할 수 있는 방향으로 나아가고자 하는 Anthropic의 철학 반영
### 클로드(Claude) AI 특징
- 윤리적 AI 원칙에 따라 설계: 사용자 요청에 대해 안전하고 책임 있는 방식으로 응답하도록 설계
- Artifacts: 대화 중 코드, 문서, 웹사이트를 실시간으로 생성하고 미리 보기
- 200K 토큰 컨텍스트 윈도우: 약 15만 단어 한 번에 처리
- Extended Thinking: 복잡한 문제를 단계별로 사고하는 심층 추론 기능

## 2) claude.ai 기본 활용
### 설정 활용
신원/역할 - 인공지능소프트웨어학과 교수
전문 분야 - ML/DL, NLP, 인공지능 교육, 프로그래밍 언어, 바이브 코딩
주요 사용 목적 - 강의 자료 및 수업 준비, 연구 논문 작성/검토, 코드 작성/디버깅, 학생 질문 단변 초안 작성, 금융/주식 투자 리서치
응답 스타일- 학술적/정확한 표현 우선, 전문 지식을 전제하고 기초 설명 생략, 또는 학생에게 설명할 수 있는 쉬운 설명
### 프로젝트
- 관련 대화 하나의 작업 공간으로 묶어서 관리
- 핵심 구성 요소
  - 지속적 메모리
    - 프로젝트 내 모든 대화에서 공유되는 System Prompt 설정 가능
    - 매 대화마다 배경 정보 반복 입력할 필요 없음
  - 파일 업로드 및 공유
    - PDF, 코드, 문서 등을 프로젝트 레벨에서 업로드하면 모든 대화에서 참조 가능
    - 논문, 강의계획서, 데이터셋 명세서 등을 한 번 업로드 후 반복 활용
  - 대화 분류 및 정리
    - 주제별로 대화를 프로젝트로 구분
    - 이전 대화 내용 추적 및 재참조 용이
### 프로젝트 지침
- 프로젝트 지침
  - 특정 프로젝트의 구체적인 맥락과 요구 사항을 이해하는데 도움
  - 일관된 맥락을 유지해야 할 때 특히 유용
  - 지침은 해당 프로젝트 내의 채팅에만 적용
- 프로젝트 지침 사용 경우
  - 프로젝트별 맥락 제공
  - 특정 워크플로우에 대한 지침 설정
  - 특정 작업 세트에 대한 요구 사항 설정
  - 클로드가 프로젝트 내에서 맡아야 할 역할이나 관점 정의
### 아티팩트(Artifacts)
- Claude가 대화창 옆 별도 패널에 독립적으로 렌더링되는 결과물을 생성하는 기능
- 지원 파일: Markdown(.md), HTML(.html), React(.jsx), SVG(.svg), Mermaid(.mermaid), PDF(.pdf)
### 아티팩트 특징
- 렌더링 패널 분리: 대화창 결과물 좌우로 분리되어 표시 -> 코드 설명과 결과 동시 확인 가능
- 반복 수정: 후속 요청으로 아티팩트 점진적 개선 가능
- 다운로드 및 복사: 생성된 결과물 바로 다운로드/클립보드 복사 가능

## 3) 스킬 개요
### 스킬
- 정의
  - 반복 가능한 작업 또는 워크플로우를 위한 구조화된 지침을 미리 등록해 자동으로 적용하는 기능
    - 특정 작업을 수행할 때 참조하는 구조화된 지식/절차 문서
    - 특정 작업 수행을 위한 규칙과 형식 정의
    - 파일 시스템에 저장되어 있다가 관련 요청이 들어올 때 클로드의 컨텍스트에 로드
    - 특정 업무의 표준 운영 절차(SOP)를 미리 주입해 두는 방법
  - 특징 및 장점
    - 좋은 프롬프트의 반복 작성에서 벗어남
      - 시간 절약 및 효율 향상
      - 토큰 비용 절감 효과
    - 일관된 출력 품질 유지가 핵심
      - 표준화된 결과 생성 가능
      - 작업 품질의 일관성 확보
### 스킬 활용
- 모든 분야
  - 다양한 파일 포멧 생성과 변환
- 교육 연구
  - 교육, 연구, 평가 등 다양한 영역에서 활용 가능
  - 강의 자료, 평가, 피드백 반복 수행
### 스킬 작동 원리
- 대화 시작
  - 스킬의 메타 데이터(name, description)만 먼저 로드
  - 관련 요청이 들어오면 전체 내용을 불러와 실행
- 점진적 로딩
  - 컨텍스트 과부하 방지
  - 활성화된 스킬만 선택적으로 사용해 응답 속도 높임
- 자연어로 워크플로우나 지침 정의
  - 코딩 없이 자연어로 커스텀 스킬 생성 가능
  - 이미 만들어진 스킬 사용 가능

### **SKILL.md**

  - YAML

    - 애플리케이션 설정 파일에 주로 사용

      - description: **트리거 조건**

    - 트리거 시 로드
    - 실제 작업 절차, 규칙, 템플릿
    - 상세 내용은 references/로 분리하고 포인터만 기재

## 4) 스킬 활용
### 스킬 생성
- 대화만으로 Skill Creator를 통해 만들거나 생성된 SKILL.md를 업로드하는 흐름 안내
- SKILL.md 구조
  - YAML frontmatter와 본문으로 구성
  - name: 스킬 이름
  - description: 자동 로드 판단 기준
  - 본문: 실제 작업 규칙과 절차
### 스킬 생성 절차
1. 반복 작업 하나 선정
    - 반복성, 규칙성, 일관성이 중요한 작업을 스킬 후보로 선정(실생활, 교육, 연구 등)
2. 스킬을 위한 프롬프트 작성
    - '스킬 목적'을 기술하면서 '추천 스킬 요청 프롬프트' 재작성 요청
3. 클로드가 질의하면 이에 대해 응답
    - 스킬 작성에 필요한 몇 개의 질의(객관식)에 응답
4. 스킬 생성 과정을 보며 기다림
    - 작업 과정 표현함
5. 생성된 스킬에서 다음 확인
    - 트리거 키워드가 description에 있는지
      - 자동 호출 되려면 claude가 자연어 요청과 스킬 설명을 잘 매칭
    - 본문에 규칙, 예시, 금지사항
      - 예시가 있으면 결과 품질 더욱 안정적
6. 수정 사항 있을 시 다시 요청
7. 생성 후 테스트
    - 자연스러운 문장으로 호출해 결과 확인
8. 필요하면 다시 요청
