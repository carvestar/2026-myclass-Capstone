## 4) ChatGPT, 진화의 여정
### OpenAI, 혁신의 중심
- OpenAI
  - 일론 머스크, 샘 올트먼, 리드 호프만 공동 창립
  - 상업적 이익보다 공공의 이익을 우선하는 것을 목표
- OpenAI 목표
  - 초기 목표
    - 인공지능 기술 민주화
    - 강력한 AI 시스템이 소수의 기업이나 정부에 의한 독점 방지
    - 비영리 연구소로 시작해 오픈 소스 소프트웨어와 연구 결과를 공개하는 방식으로 진행
- 대규모 AI 연구 비용 문제 직면
  - 2019년 영리 자회사(OpenAI LP) 설립 후 투자 유치
- OpenAI 연구 성과
  - 자연어 처리(NLP) 모델 발전 -> 핵심 프로젝트인 GPT 시리즈
  - GPT: 대량의 텍스트 데이터를 미리 학습하여 텍스트를 생성하는 변환기
### LLM 기술 용어

| 용어 | 정의 및 설명 | 예시 |
| :--- | :--- | :--- |
| **GPT (Generative Pre-trained Transformer)** | - '생성 사전 훈련 변환기' - 똑똑한 글쓰기 친구 <br> - 인터넷에 있는 수많은 글 읽고 학습하여 어떤 주제든 자연스럽게 글 작성 | GPT에게 '고양이'에 대한 글을 써달라고 하면 고양이에 대한 재미있는 이야기를 술술 씀 |
| **사전 훈련 (Pre-training)** | - GPT가 책을 통해 세상을 배우는 과정 <br> - 인터넷에 있는 수많은 텍스트 데이터를 읽으며 단어와 문장의 관계, 문법, 다양한 지식 익힘 <br> - 사전 훈련을 통해 방대한 지식 습득 후 다양한 주제에 대한 글 쓸 수 있는 능력 갖춤 | 어린아이가 책을 읽으며 세상을 배워나가는 것 |
| **미세 조정 (Fine-tuning)** | - 사전 훈련을 마친 GPT에게 맞춤형 과외 수업하는 것 <br> - 특정 분야의 데이터를 추가로 학습시켜 원하는 결과물을 더 잘 만들어내도록 함 <br> - 미세 조정을 통해 특정 분야에 특화된 능력 갖추게 됨 | GPT를 시인으로 만들고 싶다면 시를 집중적으로 학습시키고, 번역가를 만들고 싶다면 번역 데이터를 학습 |

### ChatGPT-1.0
- 2022년 출시된 GPT-3.5 기반의 챗봇
- 사용자와 자연스러운 대화 나누도록 특별히 최적화된 모델
- **인간 피드백 기반 강화 학습(RLHF) 기술 도입해 성능 향상**
  - 인간의 피드백을 통해 인공지능 모델이 학습/성능개선
- 단순한 대화혀여 AI를 넘어 코딩, 작문, 번역 등 다양한 작업 지원

### 거대 언어 모델 ChatGPT 활용 가능성
- 거대 언어 모델의 다양한 생성 분야 활용 (장점)
  - 업무 생산성 향상
  - 개인의 능력 혁신
  - 창의적 작업의 협업 -> 혁신적 결과물 작성에 기여
- 학생의 LLM 활용
  - 개인 맞춤형 학습 자료 제공
  - 방대한 정보 처리
  - 협업 및 소통 능력 강화

### ChatGPT와 함께하는 미래
- AI 혁명이 가져올 미래 사회
  - 언어 모델 발전
  - 로봇과 AI의 융합
  - AI 칩 시장 변화
  - AI의 전방위적 혁신
  - AI 윤리와 규제
- AI 미래 트렌드
  - AI 언어 모델 -> 인간 수준의 언어 이해 및 생성 능력 달성
  - 로봇공학과 AI 기술 융합 -> 인간과 유사한 로봇 일상에 더 많이 도입
  - AI 기술의 수요 증가 -> AI 칩 시장의 경쟁 심화
  - 의료, 교육, 금융 등 다양한 분야에서 혁신 주도 / 생산성 향상과 새로운 비즈니스 모델 창출
  - AI 발전과 함께 윤리적 고려와 규제 중요성 대두

## 4) 대화 질문 설계의 세계
### 대화 질문 설계 정의와 중요성
- 대화질문 설계(프롬프트 엔지니어링)
  - 생성형 AI와의 대화에서 원하는 결과를 얻기 위해 질문을 설계하고 최적화하는 과정
  - 생헝형 AI의 능력을 최대한 활용하고 사용자의 의도를 정확하게 전달하는 것이 목표
- 대화 질문 설계의 중요성과 주요 요소
  - 생성형 AI
    - 다양한 정보를 분석/종합하여 문제 해결 및 의사 결정 지원
    - 방대한 데이터를 기반으로 다양한 관점에서 문제를 분석하고 객관적 정보 제공
    - 방대 데이터 학습
  - 대화 질문 설계 중요성
    - 사용자 질문 방식에 따라 답변의 정확도와 효율성 변화
    - AI가 사용자의 의도를 정확히 파악하고 원하는 정보를 제공하는데 도움
    - 상호작용을 통해 협력적인 관계 구축 과정
  - 주요 요소
    - AI에게 역할을 부여하거나 특정 상황을 설정
    - AI의 답변에 피드백을 제공함으로써 성능 향상에 기여하고 긴밀한 협력 관계 구축
### 단계별 구체적 질문
- "Let's think step by step" 접근 방식
  - 1단계: 문제 정의
  - 2단계: 원인 분석
  - 3단계: 해결책 제안

## 5) 대화 질문의 구성 요소 탐구
### 목적과 맥락, 역할 이해하기
- 프롬프트 엔지니어링
  - AI와의 대화에서 원하는 결과를 얻기 위한 기술
  - 사용자는 프롬프트를 통해 AI의 답변을 원하는 방향으로 진행
  - 이 과정에서 목적과 맥락, 역할은 주요 구성 요소
- 질문의 **목적(Objective)**
  - AI 활용의 성공 여부를 결정짓는 핵심
  - 질문을 통해 무엇을 얻고자 하는지 구체적으로 정의
    - Ex) "한국 역사에 대해 알려 줘."보다 "조선시대 임진왜란의 배경과 결과를 요약해 줘."가 훨씬 명확한 목적 제시
- 질문의 **맥락(Context)**
  - 사용자가 필요로 하는 질문의 정보의 배경과 상황
    - Ex) "이 책에 대해 설명해 줘."보다 "최근 베스트셀러인 '세상의 마지막 기차역'의 줄거리와 등장인물을 소개해 줘."
- **역할(role, persona)**
  - AI 모델이 특정 상황에서 수행해야 하는 기능 정의
    - Ex) "네가 역사 선생님이라고 생각하고 임진왜란에 대해 설명해 줘."
### 청중과 예시, 형식 고려하기
- 효과적인 프롬프트는 청중ㅇ의 수준과 필요에 맞춰 작성
  - 질문하는 사용자(청중) 고려
  - 청중 사례
    - 대학생, 고등학생, 일반인
    - 대학생은 비판적 사고와 깊이 있는 분석을 기대하므로, 프롬프트는 단순한 정보 제공을 넘어 심층적인 이해를 돕는 방향으로 설계
- 프롬프트 엔지니어링에서 예시
  - AI 모델에게 원하는 결과물을 명확히 보여 주는 핵심 요소
  - 구체적인 예시를 제시함으로써 AI는 사용자가 원하는 답변의 형태, 내용, 스타일을 더 정확하게 파악하고 그에 맞춰 결과물 생성
- 사례
  - "한국의 유명한 시인에 대해 알려 줘." -> "윤동주의 '서시'처럼 감성적인 시를 쓰는 한국 시인 3명을 소개해 줘."
